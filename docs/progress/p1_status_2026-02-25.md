## 1) 已完成（Done）

1. 已完成 P1 基线与执行口径收敛：并发分档、路由策略、`maxInFlightTasks=32` 与 benchmark 聚合链路已落地。
2. 已完成 transfer upload/download 关键语义对齐，并补齐 fallback 安全边界与最小回归。
3. 已完成 Rust 配置项生效路径补齐：`write_timeout_ms`、连接并发限制与 Dart 初始化透传链路已接通。
4. 已完成默认公开 API 的保守化收敛与 Rust 生命周期高优先级风险修复；默认接入不再暗示 Rust 已启用。
5. 已完成一次最小公网 smoke 核对，已确认当前 benchmark / upload 基础链路可用。
6. 已修正 tracked-channel `cancelTransferTask()` 的异常语义：已知通道继续暴露真实 adapter 错误，未跟踪或 stale 场景才走双侧安全探测。
7. 已收敛 transfer 状态管理剩余 review follow-up：Dio progress 缓冲会刷新最近保留顺序，Gateway tracked task 淘汰已改为按最近活跃顺序。
8. 已固化 non-loopback benchmark 自动化入口：新增 `tool/p1_non_loopback_bench.dart`，可固定产出公网 benchmark JSON、聚合摘要与 `run_manifest.json`，并已沉淀一份“主机 -> 公网服务” smoke 样例报告。
9. 已固化 `http://47.110.52.208:7777/upload` 的命名约定、服务端回执口径与带鉴权样例归档：`remotePrefix` / `extraFields` 已写入 `run_manifest.json`，并已留存一份 `HTTP 200` 上传回执样例。

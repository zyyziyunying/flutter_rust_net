## 1) 已完成（Done）

1. 已将 Rust `DiskCache` 从占位实现升级为可用基础实现（GET 场景）。
2. 已落地缓存 key 归一化：`method + normalized_url + headers + body_hash + version`。
3. 已补齐 `TTL + ETag/Last-Modified` 条件重验证链路（`If-None-Match` / `If-Modified-Since`）。
4. 已在 `NetEngine::request` 接入缓存读取/回源/回写流程，命中新鲜缓存时返回 `from_cache=true`。
5. 已补齐命名空间治理与清理入口复用（`clear_cache(namespace)` 统一走 `DiskCache`）。
6. 已补齐 `TTL + LRU` 清理策略（默认 namespace 容量预算 64MB）。
7. 已新增 Rust 单测覆盖缓存核心行为（命中、重验证、LRU、namespace 清理与安全校验）。
8. 已完成缓存策略参数外置最小闭环：新增 `cache_default_ttl_seconds` / `cache_max_namespace_bytes`（Rust 配置 + 默认值兜底）。
9. 已打通 Dart 侧配置入口：`RustEngineInitOptions -> rust_api.NetEngineConfig -> NetEngine -> DiskCache`。
10. 已新增 Rust 单测覆盖零 TTL 行为：当 `DiskCache` 默认 TTL 为 `0` 且响应无缓存头时，缓存条目不会被当作新鲜命中；当前对外初始化配置链路中的 `cache_default_ttl_seconds=0` 仍表示“使用默认值兜底”。
11. 已完成 `engine/cache.rs -> engine/cache/` 模块化拆分，公共 API 与缓存行为保持不变。
12. 已补齐跨通道缓存契约回归（缓存关闭/开启）：新增 `test/network/cache_channel_consistency_test.dart`，以 harness 方式覆盖 `no-store` 与 `max-age` 口径下的 Dio/Rust 响应契约对齐；真实 Rust FFI 链路复测仍待后续补跑。
13. 已补齐缓存命中可观测性对齐：新增 `NetResponse.fromCache`，并将 Rust `ResponseMeta.fromCache` 透传到 Dart 统一响应模型。
14. 已新增映射回归：`test/network/rust_adapter/rust_adapter_request_test.dart` 增补 `fromCache` 字段透传断言，避免后续桥接回归。
15. 已将缓存指标接入 benchmark 报告输出：新增 `cacheHit/cacheMiss/repeatedMissCount/cacheRevalidate/cacheEvict` 字段，并补充 `--request-key-space` 复用 key 能力用于缓存收益观测；当前客户端侧 repeated miss 单独记到 `repeatedMissCount`，`cacheRevalidate/cacheEvict` 仅在本地 scenario server 口径下承载条件请求计数与重复回源信号。
16. 已对齐“缓存副本”与“响应 materialize 文件”的默认生命周期边界：Rust 普通请求的默认落盘文件已从 `cache_dir` 根目录迁移到独立 sibling 目录，`clear_cache` 清理缓存副本时不再误删待 Dart 消费的响应文件；已新增 Rust 回归覆盖“网络命中 -> 缓存命中 -> clear_cache”链路。
17. 已补跑公网 `jitter_latency` 缓存收益本地执行产物摘要（`build/remote_cache_probe_20260313/remote_jitter_cache_{cold,warm}.json`）：`cold-start` 下 Rust `cacheHit=82/96`、`reqP95=85ms`、`throughput=405 req/s`，优于 Dio `cacheHit=0/96`、`reqP95=120ms`、`throughput=186 req/s`；`warm-cache` 下 Rust `cacheHit=96/96`、`reqP95=7ms`、`throughput=1371 req/s`，显著优于 Dio `reqP95=43ms`、`throughput=226 req/s`。
18. 已确认上述历史 external 样例窗口内命中的都是 fresh TTL 路径；当前实现下若要观察 `cacheRevalidate/cacheEvict` 权威信号，应优先使用本地 scenario server 口径。
19. 已修正 `clear_cache` 的空白 namespace 风险：`clear_cache("   ")` 不再退化为“全量清理”，并已补齐 Rust/Dart 回归覆盖空白 namespace、trim 后 targeted clear 与 `cacheMaxNamespaceBytes` 配置漂移拒绝。
